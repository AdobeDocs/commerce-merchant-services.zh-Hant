# 建議型別

Adobe Commerce提供了一大組建議，您可以將這些建議部署至網站上的各個頁面。 所有建議型別都是資料導向。 它們由行為資料、產品屬性資料和量度提供支援。 為方便參考，建議型別分組如下：

- [個人化](#personalized)
- [交叉銷售和追加銷售](#crossup)
- [人氣](#popularity)
- [高效能](#highperf)

作為最佳實務，Adobe建議您在使用建議時遵循下列准則：

- 讓您的建議型別多樣化。 如果客戶一再建議相同的產品，就會開始忽略建議。

- 請勿將相同的建議部署至購物車頁面和訂單確認頁面。 考慮使用 `Most Added to Cart` （購物車頁面和） `Bought This, Bought That` 用於訂單確認頁面。

- 保持網站整潔。 請勿在同一頁面上部署超過三個建議單位。

- 如果您的商店銷售服裝， `More like this` 推薦可建議不符合所檢視產品性別的特定性別產品。 請考慮僅針對非服裝類別使用此建議型別。

## 個人化 {#personalized}

這些建議型別會根據特定購物者在您網站上的行為記錄，來建議產品。

| 型別 | 說明 |
|---|---|
| 為您推薦 | 根據每位購物者目前和先前的現場行為推薦產品。 根據購物者的瀏覽和購買記錄，顯示高度相關的建議。 此建議型別在多數購物者開始網站歷程的首頁上有效。 對於您網站上未產生任何訊號來個人化體驗的首次購物者，Adobe Commerce會根據「檢視次數最多」的建議型別顯示產品。 但是，當購物者開始與網站上的產品互動時，建議的產品會即時調整其行為。<br/><br/>**使用處：**<br/> — 首頁<br/> — 類別&#x200B;<br/><br/>**建議的標籤：**<br/>  — 專為您量身打造<br/> — 為您推薦<br/> — 因應您的購物趨勢 |
| 最近檢視的專案 | 根據瀏覽器歷史記錄，顯示購物者最近檢視的產品。 任何已刪除的產品都會由推薦單位移除。 如果沒有瀏覽器歷史記錄，或套用篩選規則時的歷史記錄不足，則不會顯示建議單位。 如果結果包含的產品少於設定的數量，建議單位只會顯示傳回的產品。<br/><br/>**使用處：**<br/> — 首頁<br/> — 類別<br/> — 產品詳細資料<br/> — 購物車<br/> — 確認&#x200B;<br/><br/>**建議的標籤：**<br/> — 最近檢視的專案<br/> — 再看一看 |

## 交叉銷售和追加銷售 {#crossup}

這些推薦型別是社交證明驅動的，可幫助購物者找到其他人喜歡的內容，或是產品驅動的，可幫助他們找到其他類似的產品

| 型別 | 說明 |
|---|---|
| 已檢視這個專案，已檢視那個專案 | 建議消費者以目前檢視的產品檢影片率不成比例的產品。<br/><br/>**使用處：**<br/> — 產品詳細資料<br/> — 購物車<br/> — 確認&#x200B;<br/><br/>**建議的標籤：**<br/> — 檢視過此產品的客戶也檢視過(PDP) |
| 已檢視此專案，但購買了其他專案 | 建議消費者在檢視目前產品後，多半會購買不合比例的產品。 協助引導購物者發現他們原本可能未注意到的產品。<br/><br/>**使用處：**<br/> — 產品詳細資料<br/> — 購物車<br/> — 確認&#x200B;<br/><br/>**建議的標籤：**<br/> — 已檢視此最終版本的客戶已購買<br/> — 客戶最終購買<br/> — 其他人檢視此產品後會購買什麼？ |
| 已購買此專案，已購買該專案 | 建議購物者經常以目前檢視的產品購買不成比例的產品。 最常用於購物車或產品詳細資料頁面，以增加相關交叉銷售產品的曝光率，從而提高平均訂單值。 顯示與購物車高度相關的產品，購物者可藉由彙總其他購物者使用目前產品所購買的專案，將其新增至購物車。<br/><br/>**使用處：**<br/> — 產品詳細資料<br/> — 購物車<br/> — 確認&#x200B;<br/><br/>**建議的標籤：**<br/> — 取得您所需的一切<br/> — 別忘了這些<br/> — 經常一起購買 |
| 更多相關資訊 | 根據類似的中繼資料（例如名稱、說明、類別指派和屬性）推薦產品。 透過評估正在檢視之產品的屬性，會推薦相同類別中的類似產品。 例如，如果購物者瀏覽瑜伽墊，建議使用裝置類別中的其他產品。 由於此建議型別無法區分性別，因此不建議服裝、時尚或其他性別特定垂直領域使用。<br/><br/>**使用處：**<br/> — 產品詳細資料<br/> — 購物車<br/> — 確認&#x200B;<br/><br/>**建議的標籤：**<br/>  — 更多這樣的產品<br/> — 類似此 |
| [視覺相似度](#visualsim) | 建議產品外觀與正在檢視的產品相似。 如果影像和產品的視覺方面對於購物體驗很重要，則此建議型別最有用。 |

## 人氣 {#popularity}

這些建議型別會建議過去七天內最受歡迎或最熱門的產品。

| 型別 | 說明 |
|---|---|
| 檢視次數最多 | 建議檢視次數最多的產品，方法是計算過去七天內發生檢視動作的工作階段數。<br/><br/>**使用處：**<br/> — 首頁<br/> — 類別<br/> — 產品詳細資料<br/> — 購物車<br/> — 確認&#x200B;<br/><br/>**建議的標籤：**<br/> — 最受歡迎<br/> — 趨勢分析<br/> — 目前熱門<br/> — 最近熱門<br/> — 受此產品啟發的熱門產品(PDP)<br/> — 最暢銷商品 |
| 購買最多 | 建議購物者過去七天內最常購買的產品。<br/><br/>**使用處：**<br/> — 首頁<br/> — 類別<br/> — 產品詳細資料<br/> — 購物車<br/> — 確認&#x200B;<br/><br/>**建議的標籤：**<br/>  — 最受歡迎<br/> — 趨勢分析<br/> — 目前熱門<br/> — 最近熱門<br/> — 受此產品啟發的熱門產品(PDP)<br/> — 最暢銷商品 |
| 加入購物車次數最多 | 建議購物者在過去七天內最常新增到購物車中的產品。 此建議型別可用於所有頁面。<br/><br/>**使用處：**<br/> — 首頁<br/> — 類別<br/> — 產品詳細資料<br/> — 購物車<br/> — 確認&#x200B;<br/><br/>**建議的標籤：**<br/>  — 最受歡迎<br/> — 趨勢分析<br/> — 目前熱門<br/> — 最近熱門<br/> — 受此產品啟發的熱門產品(PDP)<br/> — 最暢銷商品 |
| 趨勢 | 根據產品在您網站上的人氣近期動向推薦產品。<br/><br/>Adobe Sensei會彙總您網站上的瀏覽和購買資料，以判斷和排名哪些產品最近受到購物者的歡迎。 由於「趨勢」會分析最近的產品動向，因此對於營業額高的目錄來說，這是一種有效的建議型別。 如果您的目錄比較靜態，除非您對象的購物模式是高度變數，否則可能沒有用處。<br/><br/>當在首頁上使用時，Trending會推薦最近在整個網站都受歡迎的產品。 趨勢分析顯示的不是持續受歡迎的產品，而是最近受歡迎的產品。 例如，如果您有促銷特定產品的電子郵件行銷活動，電子郵件產生的熱門程度增加會增加促銷產品歸類為趨勢的可能性。<br/><br/>**使用處：**<br/> — 首頁<br/> — 類別<br/> — 產品詳細資料<br/> — 購物車<br/> — 確認&#x200B;<br/><br/>**建議的標籤：**<br/> — 趨勢分析<br/> — 目前的趨勢<br/> — 最近趨勢<br/> — 熱門產品<br/> — 趨勢相關產品(PDP) |

|---|---|
| 檢視購買轉換 | 推薦檢視與購買轉換率最高的產品。 在所有註冊了產品檢視的購物者工作階段中，最終註冊購物者購買的比例為何？<br/><br/>**使用處：**<br/> — 首頁<br/> — 類別<br/> — 產品詳細資料<br/> — 購物車<br/> — 確認&#x200B;<br/><br/>**建議的標籤：**<br/>  — 最暢銷商品<br/> — 熱門產品<br/> — 您可能對 |
| 檢視到購物車轉換 | 推薦最高檢視到購物車轉換率的產品。 在所有註冊了產品檢視的購物者工作階段中，購物者最終註冊加入購物車的比例為何？<br/><br/>**使用處：**<br/> — 首頁<br/> — 類別<br/> — 產品詳細資料<br/> — 購物車<br/> — 確認&#x200B;<br/><br/>**建議的標籤：**<br/>  — 最暢銷商品<br/> — 熱門產品<br/> — 您可能對 |
| 購買最多 | 此建議型別通常稱為「最暢銷商品」，會計算過去七天內發生下單動作的工作階段數量。 此建議型別可用於所有頁面。<br/><br/>**使用處：**<br/> — 首頁<br/> — 類別<br/> — 產品詳細資料<br/> — 購物車<br/> — 確認&#x200B;<br/><br/>**建議的標籤：**<br/>  — 最受歡迎<br/> — 趨勢分析<br/> — 目前熱門<br/> — 最近熱門<br/> — 受此產品啟發的熱門產品(PDP)<br/> — 最暢銷商品 |
| 加入購物車次數最多 | 建議購物者在過去七天內最常新增到購物車中的產品。 此建議型別可用於所有頁面。<br/><br/>**使用處：**<br/> — 首頁<br/> — 類別<br/> — 產品詳細資料<br/> — 購物車<br/> — 確認&#x200B;<br/><br/>**建議的標籤：**<br/>  — 最受歡迎<br/> — 趨勢分析<br/> — 目前熱門<br/> — 最近熱門<br/> — 受此產品啟發的熱門產品(PDP)<br/> — 最暢銷商品 |

## 視覺相似度 {#visualsim}

此 _視覺相似度_ 建議型別會推薦外觀類似產品的產品與正在檢視的產品。 當產品的影像和視覺方面是購物體驗的重要部分時，此建議型別最有用。

### 運作方式

此 _視覺相似度_ recommendation type會針對目錄中視覺上類似於目前檢視影像的其他產品提供建議。 視覺相似度包含下列方面：

- 顏色
- 形狀
- 大小
- 紋理
- 材質
- 樣式

Adobe Sensei使用AI處理和分析目錄中的影像，並建立用於判斷視覺相似性的屬性。

>[!NOTE]
>
> 如果您在非生產環境中測試此建議型別，請確定您的影像URL可公開存取。

>[!NOTE]
>
> 目前，產品影像的大小不得超過10 MB。

由於此建議型別不適用於大部分的目錄，因此預設不會啟用。 您必須明確啟用此建議型別。

### 啟用視覺相似度建議型別

>[!NOTE]
>
> 此 _視覺相似度_ 建議型別可在以下情況下使用： [安裝](install-configure.md) 作為選用模組。

1. 在 _管理員_ 側欄，前往 **行銷** > _促銷活動_ > **產品Recommendations** 以顯示 _產品Recommendations_ 儀表板。

1. 按一下 **設定** （齒輪圖示），以顯示 _設定_ 頁面。

1. 在 _Visual Recommendations_ 區段，選取 **啟用Visual Recommendations**.

1. 按一下 **儲存變更** 完成後。

   此 [建立新建議](create.md) 頁面現在顯示 **視覺相似度** 當頁面型別為時，作為可選取的建議型別 **產品詳細資料**.

啟用視覺化建議後，Adobe Sensei會啟動影像處理。 所需時間取決於目錄的大小。

### 使用處

- 產品詳細資料

### 建議的店面標籤

- 您可能也會喜歡
- 我們找到您可能喜歡的其他產品
- 受此款式啟發
